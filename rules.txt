---

Buat sebuah Sistem Informasi Akademik berbasis web yang berfungsi untuk mengelola data akademik secara komprehensif. Sistem harus memfasilitasi mahasiswa dan dosen dalam mengakses informasi akademik seperti data dosen, mahasiswa, mata kuliah, jadwal perkuliahan, dan presensi. Gunakan Bootstrap 5 melalui CDN sebagai framework CSS utama untuk styling responsif dan komponen UI, tambahkan Custom CSS untuk personalisasi tampilan, gunakan Font Awesome melalui CDN sebagai library icon, serta gunakan SweetAlert2 melalui CDN untuk alert yang interaktif. Untuk backend, gunakan Laravel Framework dengan arsitektur MVC dan gunakan Blade Templating Engine sebagai engine templating.

Bangun arsitektur templating Blade dengan struktur folder pada `resources/views/` yang terdiri dari folder `layouts` yang berisi file `app.blade.php` sebagai master layout, folder `partials` yang berisi `header.blade.php`, `sidebar.blade.php`, `footer.blade.php`, dan `navbar.blade.php`, folder `pages` yang berisi subfolder `dashboard`, `mahasiswa`, `dosen`, `mata-kuliah`, `jadwal`, `presensi`, `krs`, dan `laporan`, serta folder `auth` yang berisi `login.blade.php` dan `register.blade.php`. Terapkan konsep stack dan push dengan meletakkan `@stack('styles')` pada bagian head di `layouts/app.blade.php` untuk menampung custom CSS dari setiap halaman, gunakan `@push('styles')` pada setiap halaman untuk menambahkan CSS khusus, letakkan `@stack('scripts')` sebelum penutup body di `layouts/app.blade.php` untuk menampung JavaScript khusus, dan gunakan `@push('scripts')` pada setiap halaman untuk menambahkan script khusus.

Buat struktur database yang saling berelasi. Buat tabel dosen yang menyimpan data lengkap dosen dengan kolom nip sebagai primary key bertipe string dan unique, nama, alamat, no_hp, email yang unique, foto yang nullable, status, serta timestamp created_at dan updated_at, lalu relasikan tabel ini sebagai one to many dengan tabel pengampu. Buat tabel mahasiswa dengan kolom nim sebagai primary key string dan unique, nama, alamat, no_hp, email unique, semester, id_gol sebagai foreign key, foto nullable, status, tahun_masuk, serta timestamp, kemudian relasikan sebagai many to one dengan golongan, one to many dengan presensi_akademik, dan many to many dengan mata_kuliah melalui tabel krs. Buat tabel golongan dengan kolom id_gol sebagai primary key auto increment, nama_gol, program_studi, angkatan, dosen_wali sebagai foreign key nullable ke tabel dosen, kapasitas, serta timestamp, lalu relasikan one to many dengan mahasiswa dan jadwal_akademik.

Buat tabel mata_kuliah dengan kolom kode_mk sebagai primary key string, nama_mk, sks, semester, jenis, deskripsi nullable, silabus nullable, serta timestamp, lalu relasikan many to many dengan dosen melalui tabel pengampu, one to many dengan jadwal_akademik, one to many dengan presensi_akademik, dan many to many dengan mahasiswa melalui tabel krs. Buat tabel pengampu sebagai tabel pivot yang berisi id sebagai primary key, kode_mk sebagai foreign key, nip sebagai foreign key, tahun_akademik, semester_akademik, status, serta timestamp, dan relasikan many to one dengan mata_kuliah dan dosen. Buat tabel ruang dengan kolom id_ruang sebagai primary key, nama_ruang, gedung, lantai, kapasitas, fasilitas nullable, status, serta timestamp, dan relasikan one to many dengan jadwal_akademik. Buat tabel jadwal_akademik dengan kolom id sebagai primary key, hari, kode_mk sebagai foreign key, id_ruang sebagai foreign key, id_gol sebagai foreign key, jam_mulai, jam_selesai, tahun_akademik, semester_akademik, serta timestamp, dan relasikan many to one dengan mata_kuliah, ruang, dan golongan.

Buat tabel krs dengan kolom id sebagai primary key, nim sebagai foreign key, kode_mk sebagai foreign key, tahun_akademik, semester_akademik, status_krs, nilai_akhir nullable, nilai_angka nullable, status_lulus, serta timestamp, dan relasikan many to one dengan mahasiswa dan mata_kuliah. Buat tabel presensi_akademik dengan kolom id sebagai primary key, hari, tanggal, nim sebagai foreign key, kode_mk sebagai foreign key, status_kehadiran, jam_masuk nullable, jam_keluar nullable, keterangan nullable, pertemuan_ke, metode_presensi, serta timestamp, lalu relasikan many to one dengan mahasiswa dan mata_kuliah.

Implementasikan seluruh fitur sistem berdasarkan role yang terdiri dari admin, dosen, dan mahasiswa. Untuk admin, sediakan dashboard statistik, manajemen dosen, mahasiswa, golongan, mata kuliah, pengampu, ruangan, jadwal akademik, KRS, presensi, laporan, serta pengaturan sistem. Untuk dosen, sediakan dashboard dosen, profil dosen, mata kuliah diampu, jadwal mengajar, presensi mahasiswa, input nilai, serta laporan dosen. Untuk mahasiswa, sediakan dashboard mahasiswa, profil mahasiswa, jadwal kuliah, KRS, presensi mahasiswa, nilai akademik, dan informasi akademik.

Gunakan SweetAlert2 untuk sistem alert yang mencakup alert sukses, error, konfirmasi, info, dan warning sesuai konteks seperti penyimpanan data, validasi, penghapusan, notifikasi, atau peringatan. Sediakan fitur export laporan dalam format CSV, Excel, dan PDF dengan ketentuan format, styling, dan struktur laporan yang sesuai kebutuhan akademik. Terapkan keamanan sistem dengan autentikasi login berbasis NIM atau NIP, middleware auth, session management, role-based access control, gate dan policy, validasi input, CSRF protection, password hashing, pencegahan SQL injection, rate limiting, dan secure file upload.

Buat tampilan responsif menggunakan Bootstrap dengan penyesuaian pada berbagai breakpoint, termasuk sidebar collapse pada mobile, tabel responsif, card layout, modal full-screen, dan hamburger menu. Tambahkan fitur UX seperti loading state, spinner, skeleton loading, progress bar, toast notification, badge notifikasi, color coding status, search real-time dengan debounce, filter, date range picker, clear filter, serta pagination dengan opsi jumlah data.

Implementasikan fitur tambahan seperti QR code presensi, email notifikasi, activity log, dashboard analytics menggunakan grafik, serta kalender akademik interaktif. Ikuti workflow utama sistem yang mencakup proses KRS dari pemilihan mata kuliah hingga persetujuan, workflow presensi dari input kehadiran hingga notifikasi, serta workflow input nilai dari dosen hingga dapat dilihat mahasiswa. Gunakan struktur file Blade yang terdiri dari layouts, partials, dan pages, dengan master layout berisi CDN, stack styles, partial header, sidebar, navbar, yield content, footer, dan stack scripts.

Ikuti best practices dalam pengembangan seperti pemisahan logic di controller, penggunaan request untuk validasi, penggunaan relasi Eloquent, konsistensi naming convention, optimasi performa dengan eager loading, pagination, indexing, caching, serta menjaga maintainability melalui dokumentasi kode, formatting konsisten, reusable component, dan version control menggunakan Git. Sistem harus dibangun dengan arsitektur Blade layouts, partials, serta stack dan push agar mudah dikembangkan, dirawat, dan diperluas di masa depan.
